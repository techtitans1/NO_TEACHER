Tokenization:

Definition:

  • Tokenization is the Process of breaking down a text into smaller units called tokens.
  • Tokens can be words, subwords, characters or sentences, depending on context or its purpose,

Example: "Natural Language Processing".

  • Input:
  • Word Tokens:["Natural","Language","Processing","."]
  • Character Tokens:['N','a','t','u','r','a','l',....,'.']

Key Point: The Tokenization is the first step in any NLP Pipeline

Types Of Tokenization

1. Word Tokenization:

  • Splits text into individual words.

EX: "NLP is Awesome" -->["NLP","is","Awesome"]

2. SubWord Tokenization:

  • Breaks words into smaller meaningful units.

EX: "Unhappiness" -->["Un","happi","ness"]

3. Character Tokenization:

  • Treats Every Character as a Token.

EX: "Cat" -->["C","a","t"]

4. Sentence Tokenization:

  • Splits Paragraphs into individual sentences.

EX: "NLP is a subject. It is sub-field of AI. It is an easy subject." -->["NLP is a subject.","It is a sub-field of AI.","It is an easy subject."]

5. White Space Tokenization:

  • Splits using spaces only [Basic Method]

EX: "Hello World" -->["Hello","World"]

Advantages of Tokenization

  • It simplifies text analysis
  • Prepares text for further NLP tasks
  • Works well with Structured and un structured data
  • Essential for training machine learning models
  • Help in understand language context

DisAdvantages of Tokenization

  • Different languages require different rules
  • Token boundaries can be unclear in some languages like chinese
  • Inconsistent output across different tokenizers
  • Basic tokenizers fail with contractions

What is Contraction:

  • In Natural Language Processing (NLP), a contraction is a shortened form of one or more words that combines them using an apostrophe (').)

Examples of contractions:

  • don't ‚Üí do not
  • can't ‚Üí cannot
  • it's ‚Üí it is
  • they're ‚Üí they are
  • I'm ‚Üí I am

Uses Of Tokenization

  • Preprocessing for text classification, sentiment analysis
  • Input preparation for NLP models [BERT, GPT etc]
  • Feature Extraction in ML pipelines
  • Machine Translation
  • Question and Answering Systems
  • Search Engines

Where do we implement Tokenization practically

In NLP Pipelines:

  • An NLP Pipeline is used in many real world applications where machines need to understand or process human language
  • It helps breakdown complex text or speech into a structured format for further analysis, prediction or interaction

The real World Applications are:

  • Chatbots & Virtual Assistants
  • Search Engines
  • E-commerce Platforms & Spam Detection
  • Speech Recognition & voice Interfaces
  • Machine Translation & Social Media Analysis
  • Healthcare NLP

What Is NLP Pipeline:

  • An NLP pipeline is a series of automated steps used to convert raw human lanugage[text or speech] into a form that a computer or machine learning model can understand and process
